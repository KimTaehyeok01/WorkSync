# WorkSync Frontend 퍼블리싱 검수 및 작업 필요 항목

검수일: 2026-05-21

## 결론

현재 프론트엔드는 주요 그룹웨어 화면의 1차 퍼블리싱은 대부분 완료된 상태로 볼 수 있습니다. 라우팅, 레이아웃, 사이드바, 상단바, 대시보드, 전자결재, 업무, 게시판, 조직도, 메신저, 감사 로그, 로그인 화면이 구현되어 있으며 `npm run build`도 성공합니다.

다만 실제 서비스 기준으로는 아직 완료가 아닙니다. 대부분의 화면이 `mockData`와 컴포넌트 내부 상태에 의존하고 있고, API 연동, 인증 상태 유지, 등록/수정/삭제의 서버 반영, 파일 업로드/다운로드, 권한 처리, 반응형 검수, 예외 처리, 테스트가 남아 있습니다.

## 퍼블리싱 완료로 판단되는 범위

- 공통 앱 구조: Vite + React + React Router 기반 앱 진입점과 라우팅 구성 완료
- 공통 레이아웃: 사이드바, 상단바, 본문 영역, 풀블리드 화면 처리 구현
- 공통 UI: 카드, 버튼, 배지, 아바타, 페이지네이션, 필터바, 모달, 폼 필드 등 기본 위젯 구현
- 대시보드: 요약 카드, 최근 게시물, 결재 대기, 팀 현황, 업무 진행률 화면 구현
- 전자결재: 목록, 상세, 신규 등록 화면 구현
- 업무: 목록, 상세, 신규 등록 화면 구현
- 게시판: 목록, 상세, 글쓰기 화면 구현
- 조직도: 직원 목록, 부서 관리 모달, 직원 추가 화면 구현
- 메신저: 대화 목록, 채팅 영역, 새 대화 모달, 첨부 파일 패널 구현
- 감사 로그: 통계 카드, 카테고리/날짜/검색 필터, 로그 리스트 화면 구현
- 로그인: 사번/비밀번호 입력, 비밀번호 표시 토글, 기본 검증, 로딩 화면 구현

## 퍼블리싱 보완 필요 항목

### P0: 화면 동작 오류 또는 라우팅 불일치

- 감사 로그 페이지네이션 props 불일치 수정
  - 현재 `AuditLogPage`는 `WSPagination`에 `current`, `onChange`를 전달하지만, 공통 컴포넌트는 `page`, `onPageChange`를 기대합니다.
  - 페이지 수가 2 이상일 때 페이지네이션 표시 값과 이동 동작이 깨질 수 있습니다.

- 게시판 카테고리 라우트 반영
  - `/board/notice`, `/board/free`, `/board/qna` 라우트는 존재하지만 `BoardListPage`는 URL을 읽지 않고 내부 `category` 기본값 `all`만 사용합니다.
  - `/board/qna`는 라우트가 있지만 카테고리 옵션에는 `qna`가 없습니다.

- 업무 상세의 수정 버튼 라우트 정리
  - `TaskDetailPage`에서 `/tasks/edit/:id`로 이동하지만 라우터에는 해당 경로가 없습니다.
  - 수정 화면을 만들거나 버튼/경로를 제거해야 합니다.

- 상단바 타이틀 매핑 보완
  - 동적 상세 라우트(`/approval/:id`, `/tasks/:id`, `/board/:id`)와 게시판 카테고리 라우트의 제목/브레드크럼이 기본값으로 떨어집니다.

### P1: 퍼블리싱 완성도 보완

- 반응형 검수 확대
  - 현재 명시적인 미디어쿼리는 일부 상단바에만 확인됩니다.
  - 사이드바 고정 레이아웃, 테이블형 리스트, 상세 2컬럼, 메신저 3컬럼 화면은 모바일/태블릿 대응 검수가 필요합니다.

- 빈 상태/로딩/오류 상태 통일
  - 일부 화면은 빈 상태가 있으나 전체 도메인에 일관되게 적용되어 있지는 않습니다.
  - API 연동 전 기준으로 로딩 스켈레톤, 오류 메시지, 재시도 UI 가이드가 필요합니다.

- 텍스트와 날짜 데이터 정합성 정리
  - 대시보드와 감사 로그 등 여러 화면에 2024년/2026년 날짜가 섞여 있습니다.
  - 화면 문구, 브레드크럼, 버튼 라벨의 오타와 용어를 정리해야 합니다.

- 상세 화면 데이터 바인딩 보완
  - 전자결재 상세 화면의 문서 미리보기/결재선/첨부파일 내용이 선택된 문서와 직접 연결되어 있지 않습니다.
  - 업무 상세 화면의 본문 일부도 정적 예시 문구입니다.

### P2: UI 품질 및 유지보수

- CSS 토큰 정리: 조치 완료
  - `domains` 폴더의 CSS 사용 비중이 높은 `--ws-*` 토큰을 기준으로 삼았습니다.
  - `index.css`에서는 `tokens.css`만 전역 기준으로 import하도록 정리했습니다.
  - `theme.css`는 레거시 호환 alias만 남겨 전역 타이포/바디 스타일 중복 개입을 제거했습니다.

- 인라인 스타일 축소: 조치 완료
  - 레이아웃, 간격, 텍스트, 아이콘 색상 등 정적 인라인 스타일을 CSS Module 클래스로 이동했습니다.
  - 데이터 기반 색상/크기/진행률처럼 런타임 값이 필요한 곳은 CSS custom property 전달 방식으로 축소했습니다.

- 접근성 보완: 일부 조치 완료
  - 공통 모달에 `role="dialog"`, `aria-modal`, 제목 연결, `Escape` 닫기 처리를 추가했습니다.
  - 아이콘 버튼, 검색 지우기, 페이지네이션, 드롭다운, 파일 드롭존에 `aria-label`, `aria-expanded`, `aria-haspopup`, 키보드 조작을 보완했습니다.
  - 완전한 포커스 트랩과 외부 클릭 닫기 표준화는 추후 공통 훅으로 분리하면 좋습니다.

## 프론트엔드 기능 작업 필요 항목

### 공통/인프라

- API 클라이언트 구성
  - `axios` 의존성은 있으나 현재 API 클라이언트, 인터셉터, 에러 핸들러는 없습니다.
  - `src/api` 또는 `src/services` 계층을 만들고 도메인별 요청 함수를 분리해야 합니다.

- 인증 상태 관리
  - 로그인은 현재 입력 검증 후 홈으로 이동하는 목업입니다.
  - 토큰 저장, 로그인 유지, 로그아웃, 인증 라우트 보호, 권한별 메뉴 제어가 필요합니다.

- 서버 상태 관리 방식 결정
  - 현재는 `useState`와 `mockData`만 사용합니다.
  - React Query 또는 도메인별 커스텀 훅 등으로 목록 조회, 상세 조회, 등록/수정/삭제, 캐시 무효화를 정리해야 합니다.

- 공통 폼 검증
  - 필수값, 이메일, 전화번호, 파일 크기/확장자, 날짜 범위 등 검증 규칙이 화면별로 제각각입니다.
  - 공통 검증 메시지와 제출 비활성화 기준을 정해야 합니다.

### 로그인/권한

- 로그인 API 연동
- 토큰 만료 처리 및 자동 로그아웃
- 인증 필요 라우트 보호
- 사용자 프로필/권한 기반 사이드바 메뉴 노출 제어
- 로그아웃 버튼 실제 동작 연결

### 대시보드

- 결재 대기, 읽지 않은 메시지, 업무 완료율, 출근 현황 API 연동
- 최근 게시물/결재 대기 문서 상세 이동 연결
- 날짜를 서버 또는 현재 시간 기준으로 표시
- 대시보드 위젯 로딩/오류/빈 상태 추가

### 전자결재

- 결재 목록 조회, 검색, 상태 필터, 페이지네이션 API 연동
- 결재 상세 조회 데이터 바인딩
- 기안서 등록 API 연동
- 임시 저장 API 연동
- 결재선 추가/삭제/순서 변경 저장
- 첨부 파일 업로드/삭제/다운로드 연동
- 결재 승인/반려 API 연동 및 처리 후 상태 갱신
- 수정/삭제 권한 처리

### 업무

- 업무 목록 조회, 상태 필터, 검색, 페이지네이션 API 연동
- 업무 상세 조회 데이터 바인딩
- 업무 등록 API 연동
- 업무 수정 화면 추가 또는 수정 버튼 제거
- 업무 삭제 API 연동
- 담당자/스프린트/태그 데이터 API 연동
- 상태 변경, 진행률 변경, 댓글/첨부파일 기능 범위 확정

### 게시판

- 게시글 목록 조회, 카테고리 필터, 검색, 페이지네이션 API 연동
- URL 카테고리와 필터 상태 동기화
- 게시글 상세 조회 API 연동
- 게시글 등록 API 연동
- 게시글 수정/삭제 API 연동
- 첨부 파일 업로드/다운로드/삭제 API 연동
- 댓글 기능이 필요한지 범위 확정

### 조직도/인사

- 직원 목록 조회, 부서 필터, 검색, 페이지네이션 API 연동
- 직원 등록 API 연동
- 직원 등록 필수값 검증
- 프로필 이미지 업로드 API 연동
- 부서 목록 조회/등록/수정/삭제 API 연동
- 직급/권한 코드 데이터 API 연동

### 메신저

- 대화방 목록 조회 API 연동
- 메시지 목록 조회 및 전송 API 연동
- 새 대화 생성 API 연동
- 실시간 수신 방식 결정: WebSocket, SSE, polling 중 선택
- 읽음 처리, 안 읽은 메시지 수, 온라인 상태 연동
- 파일 첨부/다운로드 API 연동

### 감사 로그

- 로그 목록 조회, 카테고리/날짜/검색 필터, 페이지네이션 API 연동
- 통계 카드 데이터 API 연동
- CSV 내보내기 API 또는 클라이언트 생성 방식 결정
- 관리자 권한 라우트 보호

## 권장 작업 순서

1. 현재 퍼블리싱 오류 수정: 감사 로그 페이지네이션, 게시판 카테고리 라우트, 업무 수정 라우트, 상단바 동적 타이틀
2. API 클라이언트와 인증 흐름 구축
3. 로그인/라우트 보호/사용자 정보 연동
4. 목록형 화면부터 API 연동: 전자결재, 업무, 게시판, 조직도, 감사 로그
5. 등록/상세/수정/삭제 플로우 연동
6. 파일 업로드/다운로드와 메신저 실시간 기능 연동
7. 반응형, 접근성, 로딩/오류/빈 상태 QA
8. 주요 사용자 플로우 테스트 작성

## 검수 결과 요약

- 빌드 상태: 성공
- 퍼블리싱 상태: 1차 화면 구현 완료
- 서비스 완성 상태: API/인증/저장 동작 미연동으로 기능 개발 필요
- 우선 수정 필요: 라우팅 불일치와 페이지네이션 props 오류
